Docker技术入门（二）

发表于 2017-12-24

在上一篇文章中我们初步了解了Docker是干嘛用的，以及有啥好处，那在这篇文章中我们来看看Docker怎么用。

首先，需要理解Docker中几个核心的慨念。

镜像

和虚拟机里面的镜像的概念很类似，这里是指的针对Docker引擎的一种应用程序模板。比如说某个数据库的镜像文件，或者操作系统的镜像文件。

容器

容器是镜像文件运行起来的一个实例。这就意味着一个镜像能够启动多个实例。

仓库

仓库是存放镜像文件的场所，通常是在远程服务器上。仓库分为公有仓库和私有仓库两种，公有仓库中的镜像向所有人开发。Docker官方维护的仓库Docker Hub提供了超过100,000+的镜像。

让我们来做个小实验，从远程的仓库中获取一个mysql的数据库镜像，然后运行它，并且通过一个数据库的客户端软件来连接。

首先，我们要下载Docker的社区版进行安装。这里选择的安装的操作系统是CentOS 7。

$ sudo yum install docker-ce

该命令安装的是最新的Docker社区版。

安装完成后需要启动Docker。

$ sudo systemctl start docker

现在，需要从Docker的官方仓库中将我们所需要的镜像文件弄到本地来。我们可以先在官方的仓库中查一下我们所需要的镜像是否存在。

可以看到大概有我们需要的mysql官方的镜像。

将镜像文件拉到本地。

$ docker pull mysql

完成后即可启动运行一个mysql的容器，名字为test-mysql。因为是数据库的容器，所以必须设置环境变量MYSQL_ROOT_PASSWORD。

$ docker run --name=test-mysql --env="MYSQL_ROOT_PASSWORD=mypassword" mysql

启动完成后，就可以查容器实例的状态。

$ docker ps

该命令会列出所有运行的容器名。如果想列出所有不管运行还是停止的实例，后面加上参数 -a

$ docker ps -a

如果想做下过滤，可以使用参数 -f，比如：

$ docker ps -f name=mysql

想停止该实例的运行，运行STOP命令即可（执行后容器还存在）。

$ docker stop test-mysql

如果想将容器从docker引擎中移除掉，运行rm命令（执行后容器不存在）。

$ docker rm test-mysql

Docker技术入门(一)

发表于 2017-12-22

随着项目代码规模逐渐庞大，越来越多的组件和模块的出现，一方面系统的部署变得更加复杂，另外，由于项目组分属两地，异地开发的模式导致了两地的系统运行环境如何保持一致也成了问题。通过技术调研觉得使用Docker来解决我们的问题。

Docker是什么？

Docker是一家公司的名字，这家公司提供了一款同名的软件产品，是一个开源的容器引擎。那容器又是干啥的呢？顾名思义，物理世界中的容器是用来装东西的，那Docker这个容器也是用来装东西的，只不过它装的是一个个的应用程序，可以运行在众多标准的操作系统环境上，比如Linux，Windows，MacOS。

使用Docker有啥好处？

好处一：敏捷 （Agility）

在开发和部署过程中，我们经常会遇到要下载和安装各种应用软件的需要。这其实是很麻烦的一件事情。比如，有的软件在Linux环境下的安装需要各种库，如果对Linux不是很熟悉的小伙伴就会在搭环境这个环节耗时耗力。但是使用Docker，可以将安装部署的效率极大的提升。一个pull命令就可以将需要的应用软件的镜像从远程的仓库直接下载到本地，然后一个run命令就可以快速启动镜像为一个容器的实例。

另一个场景，如果需要更换我们的应用系统中的某个组件，比如想更换某个JAVA应用程序所在的JRE的版本。只需要pull所需JRE版本的镜像，停掉之前运行的JRE所在容器，将新的镜像启动起来。

好处二：可携带 （Portability）

因为是容器，所以只要有能够运行容器的环境，这个容器里的应用程序就可以在哪里都能运行。不同的操作系统，或者是运行在云平台上。

好处三：安全 （Security）

安全体现在容器与容器之间是隔离的。隔离带来的好处是每个容器在运行时所需的资源互不影响，运行时的环境变量也是独立的。

那有人就会问了，这感觉就是虚拟化的技术，和以前用的虚拟机有啥区别呢？就我目前的使用体会来讲，以前用VMware来创建虚拟机，虚拟出来的是一个操作系统环境，然后再在上面安装各种软件。创建N个虚拟机，就需要虚拟化出来N个这样的环境。每个虚拟机由于要装操作系统，所占的磁盘空间巨大，一般的就有几个G，大的几十个G也很常见。而Docker的容器都是运行在同一个操作系统上，不同容器所用的文件在操作系统的文件系统中进行了隔离。只不过给人的感觉像是各种都有一个运行的操作系统。这样的不同导致Docker启停非常快，通常几秒就搞定，而虚拟机往往需要好几分钟。Docker容器一般也就几十，或者上百M，比虚拟机小太多。
